# astrbot_plugin_LLm_search

让 AstrBot 的 LLM 人格摆脱训练数据截止日期的限制，按需联网检索实时资讯、热梗与时事；同时在持续聊天中记住用户偏好、学习用户表达风格、积累专属知识，让 AI 越来越成熟、越来越懂你。

## 功能

### 1. 联网补全现实世界认知

- **按需联网搜索**：向 LLM 注册 `web_search` 函数工具，遇到最新资讯、热梗、时事等时效性内容时自动调用。
- **结果精炼**：将搜索 API 返回的原始数据整理为 LLM 易读的条目列表，便于融入回复。
- **安全容错**：搜索失败、超时、空结果时返回友好提示，不抛出未捕获异常。

### 2. 用户档案与自动学习

- **用户档案持久化**：为每个用户保存 JSON 档案，记录已知事实、偏好与互动摘要。
- **主动事实记录工具**：向 LLM 注册 `record_user_fact` 工具，遇到值得记忆的信息时主动写入档案。
- **自动总结**：每隔若干轮对话自动提炼新的偏好/事实并更新档案。
- **人格风格自适应**：分析用户最近发言风格，在 AstrBot 原有人格基础上自动微调回复语气。

## 安装

在 AstrBot 插件管理中使用以下地址安装：

```
https://github.com/tniany/astrbot_plugin_LLm_search
```

安装后 AstrBot 会自动安装 `requirements.txt` 中的依赖。

## 配置

在 AstrBot 控制台中找到本插件配置项：

### 联网搜索

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `enabled` | 是否启用本插件 | `true` |
| `search_api_url` | 搜索 API 地址，需返回 JSON，字段包含 `results` 数组 | `""` |
| `api_key` | 搜索 API 认证密钥 | `""` |
| `max_results` | 单次搜索返回的最大结果条数 | `3` |
| `timeout` | 搜索请求超时时间（秒） | `5` |

### 用户档案与学习

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `user_profile_enabled` | 是否启用用户档案与自动学习 | `true` |
| `auto_learn_enabled` | 是否根据用户发言自动微调回复风格 | `true` |
| `max_profile_facts` | 单个用户档案最多保存的事实条数 | `50` |
| `style_window_size` | 分析用户风格时参考的最近消息条数 | `10` |
| `summary_interval` | 每隔多少轮对话自动总结一次 | `5` |
| `profile_data_dir` | 用户档案存储目录，留空使用默认路径 | `""` |

### 搜索 API 返回格式示例

```json
{
  "results": [
    {
      "title": "标题",
      "snippet": "摘要内容",
      "url": "https://example.com/article"
    }
  ]
}
```

> 如果你还没有合适的搜索 API，可以接入任何兼容上述返回格式的搜索引擎服务。

## 使用方式

启用插件后，无需手动触发指令：

- 当用户提问涉及时效性内容时，LLM 会自动调用 `web_search` 获取实时信息。
- 当 LLM 认为用户表达了值得记忆的信息时，会主动调用 `record_user_fact` 写入档案。
- 每次对话会自动记录用户风格，并在后续回复中适度微调语气。

用户档案默认保存在插件目录下的 `data/worldview_maturity/users/` 中。

## 依赖

- `aiohttp`

## 注意事项

- 请妥善保管 `api_key`，不要将其提交到公开仓库。
- 插件本身不内置搜索能力，需要自行准备或购买可联网的搜索 API。
- 用户档案仅存储在本地，不会上传到任何服务器。
