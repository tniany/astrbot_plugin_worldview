# astrbot_plugin_LLm_search

让 AstrBot 的 LLM 人格摆脱训练数据截止日期的限制，按需联网检索实时资讯、热梗与时事，补全对「地球 Online」的认知，使回复更鲜活、更贴近当下。

## 功能

- **按需联网搜索**：向 LLM 注册 `web_search` 函数工具，遇到最新资讯、热梗、时事等时效性内容时自动调用。
- **提示注入**：在每次 LLM 请求前通过 `on_llm_request` 钩子追加引导，让模型主动判断是否需要检索。
- **结果精炼**：将搜索 API 返回的原始数据整理为 LLM 易读的条目列表，便于融入回复。
- **安全容错**：搜索失败、超时、空结果时返回友好提示，不抛出未捕获异常。
- **可配置**：支持开关、搜索 API 地址、API Key、返回条数、超时时间等配置项。

## 安装

在 AstrBot 插件管理中使用以下地址安装：

```
https://github.com/tniany/astrbot_plugin_LLm_search
```

## 配置

在 AstrBot 控制台中找到本插件配置项：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `enabled` | 是否启用联网搜索 | `true` |
| `search_api_url` | 搜索 API 地址，需返回 JSON，字段包含 `results` 数组 | `""` |
| `api_key` | 搜索 API 认证密钥 | `""` |
| `max_results` | 单次搜索返回的最大结果条数 | `3` |
| `timeout` | 搜索请求超时时间（秒） | `5` |

### 搜索 API 返回格式示例

```json
{
  "results": [
    {
      "title": "标题",
      "snippet": "摘要内容",
      "url": "https://example.com/article"
    }
  ]
}
```

> 如果你还没有合适的搜索 API，可以接入任何兼容上述返回格式的搜索引擎服务。

## 使用方式

启用插件后，当用户提问涉及最新资讯、热梗、时事或网络流行内容时，LLM 会自动调用 `web_search` 工具进行检索，并将结果用于生成回复。无需手动触发指令。

## 依赖

- `aiohttp`

## 注意事项

- 请妥善保管 `api_key`，不要将其提交到公开仓库。
- 插件本身不内置搜索能力，需要自行准备或购买可联网的搜索 API。
